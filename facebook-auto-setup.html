Passo 1 — Crea l'app Meta (5 min)

  1. Vai su developers.facebook.com/apps
  2. Clicca Crea app
  3. Scegli caso d'uso: Altro → Avanti
  4. Tipo app: Business → Avanti
  5. Nome app: The Little Satoshi News Poster
  6. Email di contatto: la tua email
  7. Clicca Crea app e conferma la password Facebook se richiesto
  8. Nella dashboard dell'app, vai su Casi d'uso e aggiungi Gestire tutto sulla tua Pagina (Pages API) se disponibile

Se Meta chiede la verifica business, per i test iniziali spesso basta la modalità Sviluppo con il tuo account come tester.

Aggiungi il permesso per pubblicare (obbligatorio)

  1. Apri Casi d'uso dell'app
  2. Clicca Personalizza su Gestisci tutto sulla tua Pagina (Manage everything on your Page)
  3. Clicca Aggiungi accanto a pages_manage_posts
  4. Aggiungi anche pages_manage_read_engagement e pages_manage_metadata se disponibili
  5. Salva → torna al Graph API Explorer e rigenera il token

⚠ Non vedi pages_manage_posts in Personalizza?

Capita spesso. Meta mostra quel permesso solo se l'app ha il caso d'uso Pagina configurato correttamente. Prova in ordine:

Soluzione A — Aggiungi il caso d'uso mancante

  1. Apri la Dashboard dell'app
  2. Cerca il pulsante Aggiungi casi d'uso (o + Aggiungi)
  3. Seleziona Gestisci tutto sulla tua Pagina / Manage everything on your Page
  4. Conferma → poi clicca Personalizza su quel caso d'uso
  5. Ora dovresti vedere una lista con pages_show_list già presente e un pulsante Aggiungi per altri permessi, incluso pages_manage_posts

Soluzione B — Menu Pubblica / Permessi

In alcune versioni della dashboard Meta il permesso è qui, non in Casi d'uso:

  1. Menu sinistro → Pubblica (Publish)
  2. Clicca Gestisci tutto sulla tua Pagina
  3. Oppure vai su Revisione app → Permessi e funzioni
  4. Cerca pages_manage_posts → clicca Aggiungi o Richiedi
  5. Aggiungi anche pages_read_engagement se richiesto

Soluzione C — Ricrea l'app (se A e B non funzionano)

L'app attuale è stata creata con caso d'uso Altro. A volte Meta non espone i permessi Pagina. In quel caso:

  1. Crea una nuova app
  2. Al passo dei casi d'uso scegli direttamente Gestisci tutto sulla tua Pagina (non "Altro")
  3. Tipo: Business, nome: The Little Satoshi News Poster 2
  4. In Personalizza aggiungi pages_manage_posts
  5. Usa la nuova app nel Graph API Explorer

Cosa dovresti vedere in Personalizza (se è corretto): permessi già presenti business_management, pages_show_list, public_profile — e un elenco di permessi opzionali con pulsante Aggiungi. Se vedi solo 2–3 permessi senza possibilità di aggiungerne altri, il caso d'uso Pagina non è attivo.

Modalità Sviluppo: non serve approvazione Meta per testare su The Little Satoshi News se sei admin dell'app e della pagina. Basta aggiungere il permesso all'app e rigenerare il token.

Passo 2 — Ottieni Page ID e Token

  1. Apri Graph API Explorer
  2. In alto a destra, menu Meta App → seleziona The Little Satoshi News Poster
  3. Clicca Generate Access Token (o icona permessi)
  4. Spunta questi permessi (tutti quelli che vedi disponibili):
    • pages_manage_posts
    • pages_manage_metadata
    • pages_manage_read_engagement
    • pages_show_list
  5. Clicca Generate Access Token → accetta con Facebook
  6. Nel campo query scrivi: me/accounts
  7. Metodo: GET → clicca Submit
  8. Nella risposta JSON trova la pagina Bitcoin.Wizard1 (o il nome della tua pagina)
  9. Copia due valori:
    • "id": "123456789" → questo è il PAGE ID
    • "access_token": "EAA..." → questo è il TOKEN PAGINA

Importante: non usare il User Token. Dopo aver generato il token, nel menu sotto il campo Access Token scegli User or Page → nome della tua Pagina (es. Bitcoin.Wizard1). Copia quel token Pagina. Se me/accounts è vuoto, questo è l'unico modo per ottenerlo.

Passo 3 — Test rapido nel Graph Explorer

  1. Nel menu User or Page (sotto il token), seleziona il token della pagina Bitcoin.Wizard1
  2. Metodo: POST
  3. Query: TUO_PAGE_ID/feed (sostituisci con l'id numerico)
  4. Parametro: message = Test API The Little Satoshi News — funziona!
  5. Clicca Submit
  6. Se ricevi {"id": "..."} → l'API funziona. Controlla il post sulla pagina Facebook.

Passo 3 — File .env (test locale)

Crea il file scripts\.env:

FACEBOOK_PAGE_ID=il_tuo_id_numerico
FACEBOOK_PAGE_ACCESS_TOKEN=il_token_della_pagina

Testa la connessione:

cd C:\Users\krown\cryptofacile
python scripts\verify-facebook-token.py
python scripts\post-to-facebook.py --auto --slot 0 --dry-run

Pubblica un post di prova:

python scripts\post-to-facebook.py --auto --slot 0

Passo 4 — GitHub Actions (automatico nel cloud)

Consigliato se il sito è su GitHub Pages.

  1. Vai su Settings → Secrets → Actions
  2. Aggiungi due secrets:
    • FACEBOOK_PAGE_ID
    • FACEBOOK_PAGE_ACCESS_TOKEN
  3. Fai push del progetto su GitHub (branch main)
  4. Il workflow facebook-auto-post.yml pubblicherà automaticamente 3 volte al giorno
  5. Test manuale: Actions → Facebook Auto Post → Run workflow

Non serve tenere il PC acceso — GitHub pubblica per te.

Alternativa — Windows Task Scheduler

Se preferisci pubblicare dal tuo PC (deve restare acceso):

powershell -ExecutionPolicy Bypass -File scripts\register-facebook-tasks.ps1

Crea 3 attività: mattina 09:00, pranzo 13:00, sera 19:00.

Impostazioni piano

Modifica data/facebook-schedule.json:

{
  "startDate": "2026-06-19",
  "postsPerDay": 3,
  "timezone": "Europe/Rome",
  "posted": []
}
  • postsPerDay: 2 o 3
  • startDate: primo giorno del calendario
  • posted: si aggiorna automaticamente dopo ogni post
